# 三星堆数字祭祀身份体验系统 24 小时开发文档

## 1. 项目定位

本项目是一个结合三星堆文化元素、AI 面具生成、考古复原交互、3D/AR 佩戴、人脸表情驱动、祭祀式手势交互、古蜀身份生成和个性化桌宠的数字文博互动体验系统。

项目不只是让用户“戴上一个面具”，而是让用户通过选择纹样、复原面具、完成仪式动作，生成属于自己的“古蜀神性身份”，并在体验结束后获得一个个性化三星堆桌宠作为数字纪念品。

一句话定位：

> 这是一个将三星堆文化符号转化为可生成、可佩戴、可互动、可带走的数字祭祀身份体验系统。

## 2. 24 小时开发目标

在 24 小时内完成可演示的最小闭环版本，重点保证流程完整、视觉效果明显、创新点容易讲清楚。

最终演示流程：

1. 用户进入系统首页。
2. 用户选择三星堆特色纹样和面具残片。
3. 系统播放“考古修复式生成”动画。
4. AI 或模拟 AI 生成三星堆面具。
5. 系统根据纹样生成“古蜀神力身份”。
6. 用户进入 3D 面具预览。
7. 用户打开摄像头。
8. 系统识别人脸并将面具贴合到脸部。
9. 面具表情根据用户表情产生变化。
10. 用户通过祭祀式手势进行戴上、摘下、切换和触发特效。
11. 面具佩戴后显示文化解说卡片。
12. 体验结束后生成身份生成证书。
13. 系统根据用户身份和交互数据生成个性化桌宠。

## 3. 核心创新功能

### 3.1 面具神力属性系统

用户选择纹样时，不只是选择图案，而是在选择一种三星堆神性身份。

| 用户选择 | 系统生成身份 | 神力属性 | AR 效果 |
|---|---|---|---|
| 纵目 | 远视神使 | 洞察 | 眼部发光、视线射线 |
| 太阳纹 | 日轮祭司 | 光耀 | 背后出现金色日轮 |
| 神树纹 | 通天巫者 | 生长 | 脸旁出现树枝/青铜纹扩散 |
| 云雷纹 | 雷纹守护者 | 守护 | 面具边缘出现电光纹路 |
| 金面 | 王权化身 | 权威 | 面具浮现金色铭文 |

答辩表达：

> 我们不是简单让用户戴面具，而是通过纹样选择生成一个“古蜀身份”，让文化元素参与用户身份建构。

### 3.2 考古修复式生成交互

普通 AI 生成是“点按钮出图”，本项目将其包装为“文物复原”。

流程：

1. 屏幕出现几块青铜残片。
2. 用户选择残片纹样。
3. 残片飞入中心并开始拼合。
4. 页面显示“正在复原古蜀神面”。
5. AI 生成或模拟生成完整面具。
6. 面具从碎片状态逐渐过渡为 3D 模型。

24 小时实现方式：

- 使用 PNG 残片素材。
- 用 CSS 动画实现残片飞入、旋转、拼合。
- 后端先返回预设图片和模型 URL。
- 页面文案显示“AI 复原完成”。

### 3.3 祭祀式手势交互

将普通手势控制包装成仪式动作，提升文化叙事。

| 手势 | 仪式命名 | 功能 |
|---|---|---|
| 张开手掌 | 唤醒神面 | 戴上面具 |
| 握拳 | 封存神力 | 摘下面具 |
| 左右挥手 | 更换神相 | 切换面具 |
| 双手上举 | 祭祀降临 | 触发光效和身份强化 |

技术方案：

- 使用 MediaPipe Hands。
- 设置 1 秒触发冷却时间。
- 为每个手势配置视觉反馈和短文案。
- 保留按钮兜底，防止现场手势识别不稳定。

### 3.4 表情驱动面具动态

面具不只是贴在人脸上，而是会根据用户表情产生变化。

| 用户表情 | 面具反馈 |
|---|---|
| 张嘴 | 面具嘴部裂纹发光或下颌轻微张开 |
| 微笑 | 面具眼部亮度增强 |
| 眨眼 | 面具眼部光效闪烁 |
| 皱眉 | 面具纹样变暗或出现威严光效 |
| 头部左右转动 | 面具保持 3D 跟随旋转 |

24 小时实现方式：

- 使用 MediaPipe Face Mesh 获取嘴部、眼部和眉部关键点。
- 不做复杂面部动画骨骼，优先用光效、透明层、贴图偏移模拟表情。
- 3D 模型可使用简单 blend shape；如果时间不足，用 2D/Shader 光效替代。

### 3.5 文化解说卡片

面具佩戴后显示一张短卡片，让项目从“炫技”变成“数字文博”。

示例：

```text
纵目造型象征古蜀人对神灵视野的想象，强化了超越常人的洞察感。
```

要求：

- 每张卡片控制在 20-40 字。
- 根据用户当前面具身份显示。
- 卡片不要遮挡摄像头主画面。

### 3.6 青铜氧化动态材质

面具戴上时出现从暗铜到激活的变化。

状态：

- 初始：暗青铜。
- 唤醒瞬间：绿色铜锈纹扩散。
- 激活后：眼部、纹样或铭文微光。

实现方式：

- Three.js 修改 emissive、metalness、roughness。
- 叠加半透明纹样光效 PNG。
- 如果使用 2D 面具，使用 CSS filter、opacity、mix-blend-mode 实现。

### 3.7 身份生成证书

体验结束后生成用户专属证书。

示例：

```text
你的古蜀身份：远视神使
面具纹样：纵目 + 云雷纹
神力属性：洞察
守护色：青铜绿
仪式动作：唤醒神面 3 次
```

用途：

- 作为答辩演示收尾。
- 可截图保存。
- 可与桌宠生成结果绑定。

### 3.8 个性化桌宠

完成交互后，系统根据用户选择和交互表现生成一个个性化三星堆桌宠。

生成依据：

| 输入数据 | 影响桌宠 |
|---|---|
| 面具造型 | 决定桌宠身份 |
| 纹样选择 | 决定桌宠装饰 |
| 色彩选择 | 决定桌宠主色 |
| 手势次数 | 决定桌宠性格 |
| 是否触发特殊手势 | 决定隐藏称号或特殊光效 |

示例：

```text
用户选择：纵目 + 云雷纹 + 青铜绿
交互表现：触发唤醒神面、完成更换神相

生成桌宠：
名称：远视雷纹守护灵
身份：古蜀数字桌宠
性格：敏锐、守护、神秘
能力：洞察迷雾
```

24 小时版本：

- 使用 PNG/GIF 桌宠。
- 展示桌宠名称、性格、能力和来源纹样。
- 点击桌宠显示一句文化短句。
- 后续可扩展为桌面常驻小窗或 Live2D。

## 4. 功能范围

### 4.1 必须完成

- 首页和流程入口。
- 面具纹样/残片选择。
- 考古复原动画。
- AI 生成或模拟生成面具。
- 古蜀身份生成。
- 3D 或 2D 面具预览。
- 摄像头调用。
- 人脸识别与面具贴合。
- 至少一种表情驱动反馈。
- 至少一种手势交互。
- 文化解说卡片。
- 身份生成证书。
- 个性化桌宠生成页。

### 4.2 可以简化

- AI 生成可先用预设图模拟。
- 3D 建模不做实时生成，使用预设 GLB 模型。
- 表情驱动优先做光效变化，不做复杂骨骼动画。
- 桌宠先做静态 PNG 或 GIF。
- 后端不做数据库，使用 JSON 和静态资源。

### 4.3 不建议 24 小时内做

- 真正 AI 自动 3D 建模。
- 复杂 Live2D 桌宠。
- 多人同时识别。
- 用户登录系统。
- 大型后台管理。
- 移动端深度适配。

## 5. 推荐技术栈

前端：

- React + TypeScript
- Three.js
- MediaPipe Face Mesh
- MediaPipe Hands
- Tailwind CSS

后端：

- Node.js Express 或 Python FastAPI
- JSON 静态配置
- 本地静态资源服务

AI/模拟生成：

- 24 小时版本优先使用预设面具图片和 prompt 文案。
- 如果已有可用接口，再接入 AI 图像生成。

3D/视觉：

- GLB/glTF 面具模型。
- Three.js 渲染。
- 低面数模型和压缩贴图。

## 8. API 设计

### 8.1 获取配置项

```http
GET /api/options
```

返回：

```json
{
  "patterns": ["纵目", "太阳纹", "神树纹", "云雷纹", "金面"],
  "colors": ["青铜绿", "暗铜色", "金色"],
  "fragments": ["fragment_eye", "fragment_sun", "fragment_tree"]
}
```

### 8.2 生成面具

```http
POST /api/mask/generate
```

请求：

```json
{
  "pattern": "纵目",
  "secondaryPattern": "云雷纹",
  "color": "青铜绿",
  "fragmentIds": ["fragment_eye", "fragment_cloud"]
}
```

返回：

```json
{
  "maskId": "mask_001",
  "imageUrl": "/assets/masks/mask_001.png",
  "modelUrl": "/assets/models/mask_001.glb",
  "prompt": "三星堆纵目青铜面具，云雷纹装饰，青铜绿氧化材质"
}
```

### 8.3 生成古蜀身份

```http
POST /api/identity/generate
```

请求：

```json
{
  "maskId": "mask_001",
  "pattern": "纵目",
  "secondaryPattern": "云雷纹",
  "color": "青铜绿"
}
```

返回：

```json
{
  "identity": "远视神使",
  "power": "洞察",
  "effect": "眼部发光、视线射线",
  "description": "纵目造型象征古蜀人对神灵视野的想象。"
}
```

### 8.4 生成桌宠

```http
POST /api/pet/generate
```

请求：

```json
{
  "maskId": "mask_001",
  "identity": "远视神使",
  "power": "洞察",
  "pattern": "纵目",
  "color": "青铜绿",
  "gestureStats": {
    "wakeCount": 3,
    "switchCount": 1,
    "specialGesture": true
  },
  "expressionStats": {
    "smileTriggered": true,
    "mouthOpenTriggered": true
  }
}
```

返回：

```json
{
  "petId": "pet_001",
  "name": "远视雷纹守护灵",
  "identity": "古蜀数字桌宠",
  "personality": ["敏锐", "守护", "神秘"],
  "ability": "洞察迷雾",
  "imageUrl": "/assets/pets/pet_001.png",
  "cardText": "你唤醒了远视雷纹守护灵，它象征洞察、守护与青铜神力。"
}
```

## 13. 答辩讲解重点

可以按 5 个创新点讲：

1. 文化身份化：纹样不只是装饰，而是生成古蜀身份。
2. 生成叙事化：AI 生成被包装成考古复原过程。
3. 交互仪式化：手势不只是控制，而是祭祀动作。
4. 面具生命化：面具可以根据人的表情产生回应。
